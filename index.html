2.Maths

2.1.Calculator

var
Variables are defined by var following the name of the variable, var new = 0 creates a variable named "new" that has the value "0". Var is global and local,can be updated, re-declared and hoisted while being initialized as undefined.

let
Let has most of the functions "var" has. "Let" is local, can be updated, can NOT be re-declared and can be hoisted but won't be initialized.

const
Const looks more like let then it does var. It is local, can NOT be updated, can NOT be re-declared and can be hoisted but won't be initialized.

document
"document" refers to the entire HTML document

.getElementById("exampleID")
When used with document.getElementById it will search the entire HTML document for tags with id="example" and fetches the entire element.

.value
When used with document.getElementById() it will fetch the value of the element with the defined "id".

On hoisting:
Hoisting is the practice of declaring before initializing.



var y;
This is declaring, this should happen at the top of a scope.



var y = 3
 This is initializing.



    var y;
    elem = document.getElementById("demo");
    elem.innerHTML = x + " " + y;
    y = 3

 First "y" is declared, this is no problem since we are dealing with a "var", when the variable is initialized after the bit of code it get's hoisted and the block of code runs as if it always knew that y = 3 from the start.
